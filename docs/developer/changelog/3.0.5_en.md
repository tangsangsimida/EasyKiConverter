# Changelog — v3.0.5

**Release date**: 2026-02-12

> See [CHANGELOG.md](../CHANGELOG.md)

## [3.0.5] - 2026-02-12

### Fixes

- **MainWindow modular refactoring**
  - Split `MainWindow` inline UI logic into independent high-performance components: `TitleBar`, `HeaderSection`, `ComponentInputCard`, `BomImportCard`, `ComponentListCard`, `ExportSettingsCard`, `ExportProgressCard`, `ExportResultsCard`, `ExportStatisticsCard`, `ExportButtonsSection`, etc.
  - Significantly improved QML code maintainability and rendering efficiency through component decoupling.
  - Code location: `src/ui/qml/components/`

- **Core import/export module single responsibility refactoring**
  - Split bloated `EasyedaImporter` and `Exporter` classes into specialized classes.
  - Added `EasyedaSymbolImporter` (symbol import), `EasyedaFootprintImporter` (footprint import), `EasyedaUtils` (common utilities).
  - Reduced core logic code volume, improved testability.
  - Code location: `src/core/`

- **BOM parsing library and service refactoring**
  - Added `BomParser` module, optimized BOM file import process.
  - Implemented export pipeline cancellation mechanism and thread pool configuration optimization, reducing parsing failure rate.
  - Code location: `src/services/`

- **Weak network P0-1: NetworkWorker timeout mechanism**
  - Added `executeRequest` generic network request method, unified handling of timeout, retry and error recovery.
  - All 4 fetch methods (component info, CAD data, 3D model OBJ/MTL) added QTimer timeout protection.
  - Default timeout 30s (component info/CAD data), 3D model timeout 45s.
  - Eliminated risk of `QEventLoop::exec()` permanently blocking threads in weak network environments.
  - Code location: `src/workers/NetworkWorker.h`, `src/workers/NetworkWorker.cpp`

- **Weak network P0-2: FetchWorker allows retry after timeout**
  - Removed `retryCount = MAX_HTTP_RETRIES + 1` skip logic.
  - Timeout (OperationCanceledError) now triggers normal retry flow.
  - In weak network environments, timeout is the most common error type; this fix significantly improves success rate.
  - Code location: `src/workers/FetchWorker.cpp`

- **Weak network P0-3: Increased FetchWorker timeout duration**
  - Component info timeout: 8s -> 15s.
  - 3D model timeout: 10s -> 30s.
  - STEP model timeout: 10s -> 30s (no longer hardcoded, unified using `MODEL_3D_TIMEOUT_MS` constant).
  - Code location: `src/workers/FetchWorker.h`

- **Weak network P1-4: Rate limit backoff changed to true exponential backoff**
  - Backoff strategy changed from `+1000` (linear) to `*2` (exponential), initial 1s, max 8s.
  - More effectively handles API rate limiting.
  - Code location: `src/workers/FetchWorker.cpp`

- **Weak network P1-5: Increasing retry delay**
  - Retry delay changed from fixed 500ms to increasing delay: 3s / 5s / 10s.
  - Referenced mature NetworkUtils strategy to give server more time to recover.
  - Code location: `src/workers/FetchWorker.h`, `src/workers/FetchWorker.cpp`

- **Weak network P1-6: Added random jitter to retry delay**
  - Added `calculateRetryDelay` helper method.
  - Added +/-20% random jitter on base delay, effectively mitigating thundering herd effect.
  - Both FetchWorker and NetworkWorker use jittered increasing delay strategy.
  - Code location: `src/workers/FetchWorker.h`, `src/workers/FetchWorker.cpp`

- **Weak network P1-7: Fixed thread_local QNAM memory leak**
  - Registered `QThread::finished` cleanup callback for `thread_local QNetworkAccessManager*`.
  - Ensured QNAM is correctly deleted when thread pool threads are recycled, not permanently leaked.
  - Code location: `src/workers/FetchWorker.cpp`

- **3D model export failure no longer blocks symbol/footprint export**
  - `writeSuccess` determination changed from "symbol + footprint + 3D model all succeeded" to "symbol + footprint core output succeeded".
  - 3D model export failure only logs a warning, does not affect normal export and merge of symbols and footprints.
  - Fixed the issue where 3D model download failure caused already-parsed symbols and footprints to not be written.
  - Code location: `src/workers/WriteWorker.cpp`

### Improvements

- **NetworkWorker code refactoring**
  - Extracted `executeRequest` generic method, eliminated significant duplicate code across four fetch methods.
  - Unified timeout, retry, progress reporting and error handling logic.
  - Code lines reduced from ~430 to ~280, maintainability significantly improved.

- **Build system optimization**
  - Enabled `compile_commands.json` generation, providing better IDE/LSP (e.g., VS Code) intellisense support.
  - Unified all project source file encoding to UTF-8 (No BOM).

- **Test framework integration**
  - Introduced QtTest test framework, supporting unit tests, integration tests, benchmark tests and manual tests.
  - Added `tests/` directory structure, containing unit tests (EasyEDA API, data models, ViewModels), integration tests, benchmark tests and UI tests.
  - Implemented dependency injection pattern, using `INetworkAdapter` interface to support mock network requests.
  - Added coverage support option (GCC/MinGW), can be enabled via `-DENABLE_COVERAGE=ON`.
  - Code location: `tests/`, `src/core/utils/INetworkAdapter.h`, `CMakeLists.txt`

- **Developer tools expansion**
  - Added `tools/python/manage_version.py` version management tool, automatically synchronizes version info in vcpkg.json, CMakeLists.txt and src/main.cpp.
  - Added `tools/python/analyze_lines.py` code line count analysis tool.
  - Added `tools/python/convert_to_utf8.py` file encoding conversion tool.
  - Added `tools/python/fix_qml_translations.py` QML translation fix tool.
  - Improved `tools/README.md` and `tools/README_en.md` tool documentation, added detailed usage instructions and environment requirements.
  - Added detailed header comments and documentation to `tools/windows/format_code.bat` and `tools/python/manage_version.py`.
  - Code location: `tools/`

- **Documentation system improvement**
  - Added Architecture Decision Record (ADR) system, documenting important architectural decisions.
  - Added testing guide documentation (`docs/developer/TESTING_GUIDE.md`).
  - Added performance baseline documentation (`docs/developer/performance_baseline.md`).
  - Updated architecture documentation, adding system architecture diagrams and workflow diagrams (Excalidraw format).
  - Code location: `docs/project/adr/`, `docs/developer/`, `docs/diagrams/`
