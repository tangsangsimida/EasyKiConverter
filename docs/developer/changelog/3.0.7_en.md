# Changelog — v3.0.7

**Release date**: 2026-02-15

> See [CHANGELOG.md](../CHANGELOG.md)

## [3.0.7] - 2026-02-15

### Fixes

- **Fixed SIGSEGV crash on application exit**
  - Refactored object lifecycle management in `main.cpp`. Business objects (ViewModels and Services) are no longer parented to `app`, changed to manual lifecycle management.
  - Implemented explicit "reverse-order destruction" mechanism: first destroy UI-related ViewModels, then backend Services, finally clean up the logging system.
  - Removed unstable `QThreadPool::waitForDone` call in `ComponentListViewModel` destructor.
  - Ensured all background threads (including async log threads and model generation threads) safely stop before Qt core infrastructure is destroyed.
  - Fixed `QtLogAdapter` uninstallation order to prevent exit-stage internal messages from triggering an already-closed logging system.

### Improvements

- **Improved architecture transparency**
  - Added ADR-008, clarifying the explicit lifecycle management strategy for key business objects.
  - Replaced blocking thread waits with `QPointer` safe mode, improving system reliability.
